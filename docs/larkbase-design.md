# LarkBase設計書：就労継続支援B型開業プロジェクト管理

## 概要

合同会社LIVENOW様の就労継続支援B型開業プロジェクト管理をLarkBaseで構築します。
現行のGoogleスプレッドシート（Excel形式）から、より効率的なプロジェクト管理が可能な構造へ再設計します。

---

## 現行Excel構造の分析

### シート一覧
1. **開業スケジュール** - メインWBS（72行、8カテゴリ）
2. **指定申請書類状況一覧** - 申請書類管理
3. **面接者情報** - 採用候補者管理
4. **提供書類一覧** - ARU⇔LIVENOW間の書類受け渡し
5. **物件リスト** - 物件候補管理（29件）
6. **不動産リスト** - 不動産会社連絡先
7. **議事録シート** - 打ち合わせ記録（3件）

### 現行の課題
- WBS番号とタスクが同一シートで階層構造が不明瞭
- 日付がExcelシリアル値（45931等）で可読性が低い
- シート間のデータ連携がない
- ステータス管理が簡素（未/完了のみ）
- 進捗の可視化機能がない

## ビュー設計

### タスクテーブルのビュー

1. **全タスク一覧（テーブルビュー）**
   - 全タスクを表示
   - カテゴリ、ステータスでフィルタ可能

2. **ガントチャート（タイムラインビュー）**
   - 開始日〜期限を横軸に表示
   - カテゴリごとにグループ化

3. **カンバンボード（カンバンビュー）**
   - ステータスで列分け
   - ドラッグ&ドロップでステータス変更

4. **ARU担当タスク（フィルタビュー）**
   - 担当者 = ARU でフィルタ

5. **LIVENOW担当タスク（フィルタビュー）**
   - 担当者 = 貴社 でフィルタ

6. **今週のタスク（フィルタビュー）**
   - 期限が今週内のタスク

### 物件テーブルのビュー

1. **全物件一覧**
2. **就労B使用可能物件** - ステータス = 就労B使用可能
3. **確認中物件** - ステータス = 確認中
4. **地図ビュー** - 住所で地図表示

## リレーション図

```
プロジェクト情報
    │
    └── WBSカテゴリ (1:N)
            │
            └── タスク (1:N)
                    │
                    ├── 申請書類 (N:M)
                    └── 議事録 (N:M)

物件 ──── 不動産会社 (N:1)

面接者（独立）
提供書類（独立）
```
